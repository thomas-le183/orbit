# Task `estimatedTime` Field Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `estimatedTime` a real, persisted task field end-to-end (DB → API → shared DTO → frontend), wire it into the scheduler, persist bar-resize edits, and delete the hardcoded mock timeline data.

**Architecture:** Add a nullable `estimated_time` integer (minutes) column to the `task` table, thread it through the shared Zod DTO and the NestJS service, expose it on the frontend `Task` type, map it onto `TimelineItem`, and persist scheduler resize edits via the existing `useUpdateTask` PATCH path. Then remove the `timelineItems` mock array, its seed test, the no-`projectId` context fallback, and the unlinked `/timeline` route.

**Tech Stack:** Drizzle ORM + PostgreSQL, NestJS, Zod (`@orbit/shared`), React 19 + TanStack Query, Vitest (web), Jest (api).

## Global Constraints

- `estimatedTime` is **minutes**, non-negative integer, nullable (a task may have no estimate).
- Run all `pnpm` commands from the stated directory (repo root unless noted).
- Never hand-edit `apps/web/src/routeTree.gen.ts` — it is regenerated by the TanStack Router Vite plugin.
- Do not write raw SQL by hand for schema changes — generate migrations via `pnpm db:generate`.
- Keep the two better-auth instances in sync (not touched by this plan, but a repo rule).
